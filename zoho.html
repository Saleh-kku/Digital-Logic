System Architecture

High-Level Architecture

Data Flow

The system implements bidirectional data flow between Zoho Creator and Zoho Books:

  • Creator → Books: Client information, invoice details, payment data
  • Books → Creator: Financial status, invoice IDs, payment confirmations
  • Real-time Sync: Using Zoho API and webhooks for immediate updates
  • Data Validation: Implemented at both Creator and Books levels
  • Error Handling: Failed transactions are logged and can be retried

Integration Components

  • Zoho Creator API Functions: Custom Deluge scripts to handle data transfer
  • Zoho Books API Endpoints: Clients, Invoices, Items, Payments
  • Authentication: OAuth 2.0 for secure API access
  • PDF Generation: Using zoho.creator.createPDF() for receipts
  • Scheduled Workflows: Automated payment reminders and reconciliation
  • Error Recovery: Intelligent retry mechanism with error logs

Payment Processing Architecture

Cash Payment

  1. Sales records transaction
  2. Status: "Cash Pending Deposit"
  3. Receipt generated for client
  4. Admin confirms deposit
  5. Books records completed payment

POS Payment

  1. Sales enters POS reference #
  2. Status: "POS Pending Confirmation"
  3. Receipt generated for client
  4. Finance verifies POS transaction
  5. Books records completed payment

Bank Transfer

  1. Sales enters bank reference
  2. Optional proof upload
  3. Status: "Bank Pending Confirmation"
  4. Finance verifies bank deposit
  5. Books records completed payment

Deferred Payment

  1. Sales sets due date & interest
  2. Status: "Awaiting Approval"
  3. Manager reviews & approves
  4. Status: "Deferred - Pending Payment"
  5. Automated reminders sent
  6. Finance confirms when paid

Security & Audit Features

Transaction Logging
  • Every payment action is timestamped

Permissions Control
  • Role-based access to sensitive functions

Error Handling

Business Workflows

Immediate Payment Workflow (Cash/POS/Bank)

Process Steps:

  1. Client Selection: Sales team selects existing client or adds a new one
  2. Invoice Creation: Sales creates invoice with product details and amounts
  3. Payment Processing:
    • Cash: Record amount received, mark as "Cash Pending Deposit"
    • POS: Enter POS reference number, mark as "POS Pending Confirmation"
    • Bank Transfer: Enter transfer details, optional proof upload, mark as "Bank Pending Confirmation"
  4. Receipt Generation: System generates PDF receipt for customer
  5. Finance Confirmation: Finance department confirms payment receipt:
    • Verifies cash deposit, POS transaction, or bank transfer
    • Updates status to "Payment Confirmed"
    • Records confirmation date and user who confirmed
  6. Books Integration: Invoice and payment details automatically transfer to Zoho Books

Deferred Payment Process

Process Steps:

  1. Client Selection: Sales team selects existing client or adds a new one
  2. Invoice Creation: Sales creates initial invoice with product details
  3. Deferred Payment Setup: Sales specifies:
    • Due date for payment
    • Interest rate (if applicable)
    • Any additional payment notes
  4. Approval Submission: System sets status to "Awaiting Approval" and notifies manager
  5. Manager Review: Manager reviews client history and payment terms
  6. Approval Process:
    • If Approved: System creates invoice in Zoho Books marked as "Deferred - Pending Payment"
    • If Rejected: Manager provides reason, sent back to sales for revision
  7. Reminder Setup: System automatically schedules payment reminders:
    • 7 days before due date
    • 1 day before due date
    • On due date
    • Follow-up reminders if payment is late
  8. Payment Monitoring: Finance team monitors due payments
  9. Payment Confirmation: Once received, finance confirms payment similar to immediate payment workflow

Deluge Script Components

On Form Submit

// Check if client exists in Zoho Books
clientInfo = zoho.books.getRecords("contacts","organization_id");
clientExists = false;

// Search for matching client
foreach(client in clientInfo)
{
    if(client.get("contact_name") == input.Client_Name)
    {
        clientExists = true;
        input.Client_Books_ID = client.get("contact_id");
        break;
    }
}

// Create client if doesn't exist
if(!clientExists)
{
    newClient = Map();
    newClient.put("contact_name",input.Client_Name);
    newClient.put("phone",input.Phone_Number);
    // Add other client fields
    
    clientResponse = zoho.books.create("contacts",newClient,"organization_id");
    if(clientResponse.get("code") == 0)
    {
        input.Client_Books_ID = clientResponse.get("contact").get("contact_id");
    }
    else
    {
        input.Error_Log = "Client creation error: " + clientResponse.get("message");
        input.Payment_Status = "Error -- Needs Fix";
        return input;
    }
}

// Process based on payment type
if(input.Payment_Type != "Deferred")
{
    // Create invoice immediately
    invoiceMap = constructInvoiceData();
    invoiceResponse = zoho.books.create("invoices",invoiceMap,"organization_id");
    
    if(invoiceResponse.get("code") == 0)
    {
        input.Invoice_Books_ID = invoiceResponse.get("invoice").get("invoice_id");
        
        // Set appropriate status based on payment type
        if(input.Payment_Type == "Cash")
        {
            input.Payment_Status = "Cash Pending Deposit";
        }
        else if(input.Payment_Type == "POS")
        {
            input.Payment_Status = "POS Pending Confirmation";
        }
        else if(input.Payment_Type == "Bank Transfer")
        {
            input.Payment_Status = "Bank Pending Confirmation";
        }
    }
    else
    {
        input.Error_Log = "Invoice creation error: " + invoiceResponse.get("message");
        input.Payment_Status = "Error -- Needs Fix";
    }
}
else
{
    // For deferred payment
    input.Payment_Status = "Awaiting Approval";
}

return input;

Error Handling & Recovery

Error Logging

All errors are captured in the Error_Log field with:

  • Timestamp of error
  • Error type and message
  • Function where error occurred
  • User who was performing the action

Sample error log entry:

[15-Dec-2023 14:32:45] API Error in createInvoice: 
Authentication failed. Invalid API token.
User: john.smith@company.com

Recovery Mechanism

For records with "Error -- Needs Fix" status:

  1. Admin dashboard shows all error records
  2. "Retry" button attempts the failed operation again
  3. "Fix and Retry" allows editing problem fields before retry
  4. System maintains full audit trail of retry attempts

Common API Errors & Handling

Error Type Handling Strategy
Authentication Failure Notify admin, request token refresh
Rate Limiting Automatic retry with exponential backoff
Duplicate Client Search and link existing record instead
Invalid Data Highlight fields needing correction
Network Timeout Automatic retry up to 3 times

Daily Monitoring

Automated daily checks:

  • Identify stuck records (no status change in 24+ hours)
  • Alert for records with retry failures
  • Data reconciliation between Creator and Books
  • Summary report emailed to administrators
